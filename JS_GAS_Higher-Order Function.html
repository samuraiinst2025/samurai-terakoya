高階関数・コールバック・map/filter/reduce 完全ガイド

高階関数（Higher-Order Function）とは

高階関数とは、次のどちらかを満たす関数です。

  1. 関数を引数として受け取る
  2. 関数を返り値として返す

基本例：map

numbers.map(n => n * 2)

map は関数を引数として受け取るため、高階関数です。

構造イメージ

numbers ↓ map（高階関数） ↓ callback（コールバック関数）

JavaScriptの関数の基本から整理する

STEP 1：普通の関数

function add(a, b) {
  return a + b;
}
add(2, 3);

STEP 2：JavaScriptでは「関数 = 値」

JavaScriptでは、関数は値として扱えるため、以下のような操作が可能です：

  • 変数に入れられる
  • 引数として渡せる
  • 返り値にできる
const add = function(a, b) {
  return a + b;
};

STEP 3：関数を引数として渡す

function run(func) {
  func();
}

run(function() {
  console.log("実行された");
});

ここで渡している function(){ ... } がコールバック関数です。

STEP 4：コールバック関数とは

他の関数に渡して、あとで実行される関数 
run(function() {
  console.log("処理");
});
// function(){ console.log("処理") } がコールバック

STEP 5：高階関数

関数を受け取る関数が高階関数（Higher-Order Function）です。

numbers.map(n => n * 2)
// map = 高階関数
// n => n * 2 = コールバック関数

mapの内部構造

map は概念的に以下のように動いています：

function map(array, callback) {
  const result = [];
  for (const item of array) {
    result.push(callback(item));
  }
  return result;
}

callback(item) が要素ごとに実行されるのがポイントです。

コールバックの書き方2種類

function式

numbers.map(function(n) {
  return n * 2;
});

アロー関数（短く書ける）

numbers.map(n => n * 2);
アロー関数 = コールバックを短く書くための書き方

用語の関係まとめ

用語 意味
高階関数 関数を受け取る関数（例：map）
コールバック関数 渡される関数（例：n => n * 2）
アロー関数 コールバックを短く書く書き方
高階関数 ↓ コールバック関数 ↓ 関数の書き方（function / アロー関数）

一言まとめ（講座向け）

高階関数 = 関数を受け取る関数
コールバック関数 = 渡される関数

概念 たとえ

Python

クラスメソッドとコールバックの違い

クラスメソッドの結果を変数に入れることとの比較：

クラスメソッドの場合

const sheet = SpreadsheetApp.getActiveSheet();
// メソッド実行 → 値を返す → 変数に入れる

mapのコールバックの場合

numbers.map(n => n * 2)
// 関数を渡す → mapが何度も実行 → 結果を配列にまとめる
メソッド コールバック
動き メソッド → 値を返す 関数を渡す → 後で実行される
例 getActiveSheet() n => n * 2

共通点：JavaScriptでは「関数 = 値」なので、どちらも変数や引数として扱える。

mapのコールバックにメソッドを渡す例

strings.map(s => s.toUpperCase())
numbers.map(Math.sqrt)  // n => Math.sqrt(n) と同じ

配列処理の基本3つ：map / filter / reduce

これらはすべて 高階関数 + コールバック関数 という同じ構造で動きます。

1. map：配列を変換する

役割：配列の各要素を変換する

const numbers = [1, 2, 3];
const result = numbers.map(n => n * 2);
// → [2, 4, 6]
[1, 2, 3] ↓ map n → n * 2 ↓ [2, 4, 6]
🏭 イメージ：材料 → 加工 → 新しい材料

2. filter：条件に合う要素だけ残す

役割：条件に合う要素だけを選別する

const numbers = [1, 2, 3, 4, 5];
const result = numbers.filter(n => n % 2 === 0);
// → [2, 4]
[1, 2, 3, 4, 5] ↓ filter n が偶数？ ↓ [2, 4]
🔍 イメージ：材料 → 選別 → 必要なものだけ

3. reduce：配列を1つの値にまとめる

役割：配列を1つの値に集約する

const numbers = [1, 2, 3, 4];
const sum = numbers.reduce((acc, n) => acc + n, 0);
// → 10
[1, 2, 3, 4] 0 + 1 → 1 1 + 2 → 3 3 + 3 → 6 6 + 4 → 10
📦 イメージ：材料 → 集約 → 1つの結果

3つの違いまとめ

関数 役割 結果
map 変換 配列（同じ要素数）
filter 選別 配列（要素数が減る場合も）
reduce 集約 単一の値

GASでよく出る例

// 各行の1列目を取り出す
const values = rows.map(row => row[0]);

関数型プログラミングとの関係

これらの仕組みは関数型プログラミングというスタイルの中心概念です。

なぜ「関数型」と呼ばれるのか

関数型プログラミングでは、関数を値として扱うことで以下のような利点が生まれます：

  • 関数の再利用性が高まる — 処理を関数として切り出し、様々な場面で再利用できる
  • コードの意図が明確になる — mapは「変換」、filterは「選別」と、目的が一目で分かる
  • 副作用を減らせる — 元の配列を変更せず、新しい配列を返すため安全

💡 ポイント：forループとの違い

従来のforループでは「どうやって処理するか」（手続き）を書きますが、map/filter/reduceでは「何をしたいか」（目的）を書きます。これにより、コードが読みやすく、保守しやすくなります。

高階関数 ↓ 関数を受け取る ↓ コールバックを実行

実はこの3つは、forループを抽象化したものでもあります：

for ループ ↓（抽象化） map / filter / reduce

具体例で比較

従来のforループ（手続き型）

// 配列を2倍にする処理
const numbers = [1, 2, 3];
const result = [];
for (let i = 0; i < numbers.length; i++) {
  result.push(numbers[i] * 2);
}
// 「どうやって処理するか」を細かく指定している

mapを使った方法（関数型）

// 配列を2倍にする処理
const numbers = [1, 2, 3];
const result = numbers.map(n => n * 2);
// 「2倍にしたい」という目的だけを表現している

全体の関係図

高階関数 ↓ コールバック関数 ↓ 関数の書き方 ・function ・アロー関数

最終まとめ

高階関数 = 関数を受け取る関数
コールバック関数 = 渡される関数
アロー関数 = コールバックを書くための短い書き方

具体例で確認

const numbers = [1, 2, 3];
numbers.map(n => n * 2);

// numbers → map（高階関数）→ n => n*2（コールバック）→ [2, 4, 6]
