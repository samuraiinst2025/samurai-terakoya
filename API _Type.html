APIの種類を理解しよう

初学者向けガイド

「APIってよく聞くけれど、色々な文脈で使われていてよく分からない…」
このドキュメントは、そんな初学者の方に向けて、APIの広い意味と、代表的な3つの種類（区分）を分かりやすくまとめたガイドです。

0. そもそもAPIとは？（広義の定義）

APIとは「Application Programming Interface」の略です。

  • Application（アプリ）：ソフトウェアや機能のこと
  • Interface（インターフェース）：何かと何かをつなぐ「窓口」や「接点」のこと

つまり、広義のAPIとは「外部の機能を呼び出して使うための、あらゆる手段や窓口」を指します。日常会話やビジネスの場では「Web API（インターネットを使ったHTTP通信）」を指すことが多いですが、プログラミングの世界ではより広い意味で使われます。本資料では、この「広義のAPI」として全体を整理します。

🍽️ レストランのメニューに例えると…
APIはレストランの「メニュー表」のようなものです。厨房の中でシェフがどのように料理を作っているか（複雑な内部処理）を知らなくても、メニューから「オムライス」を注文（APIを呼び出し）すれば、完成した料理（結果）を受け取ることができます。

1. 区分① ライブラリ/SDK型API

設定不要・即使える

概念：自分のコードの中にインポート（読み込み）して使う関数の集まりです。最初から環境に用意されている標準装備のものや、追加でインストールして使うものがあります。

  • 特徴：HTTP通信（インターネット通信）が不要で、コード内だけで完結します。
  • メリット：設定が少なく、最も手軽に呼び出すことができます。
Python の例
# pandasライブラリを読み込んで機能（API）を使う
import pandas as pd

df = pd.read_csv("data.csv")  # CSVファイルを読み込む
print(df.head())              # 最初の5行を表示
GAS（Google Apps Script）の例
// ※ GASの組み込みサービスはimport宣言も不要！最初から使えます
const sheet = SpreadsheetApp.getActiveSheet();
const value = sheet.getRange("A1").getValue();

いつ使うか：そのプログラミング環境（言語やプラットフォーム）にすでに用意されている標準機能を使いたいとき。何かやりたいことがあったら、まず最初にこの区分で解決できないか確認するのが基本です。

2. 区分② 外部API型（REST API / HTTP API）

インターネット越しに機能を借りる

概念：HTTPリクエストという通信手段を使って、インターネット越しに外部サービス（他社）の機能を呼び出す形式です。一般的に「API連携」と呼ばれるものの多くはこれに該当します。

  • 特徴：呼び出しには認証（APIキーやOAuthなど）が必要です。
  • ルール：相手が公開しているAPI仕様書（リファレンス）に従って、GETやPOSTなどのメソッドを使って通信します。
自分のコード ── [HTTP リクエストを送信] ──▶ 外部サービス
自分のコード ◀── [結果（レスポンス）を返却] ── 外部サービス
GAS の例：Slack APIにメッセージを送信する
const url = "https://slack.com/api/chat.postMessage";
const options = {
  method: "post",
  headers: {
    "Authorization": "Bearer YOUR_TOKEN"
  },
  payload: JSON.stringify({
    channel: "#general",
    text: "こんにちは！GASから送信しました🎉"
  })
};
UrlFetchApp.fetch(url, options);  // ← HTTPリクエストを送る
Python の例：OpenAI APIを呼び出す（簡易）
import requests

response = requests.post(
  "https://api.openai.com/v1/chat/completions",
  headers={"Authorization": "Bearer YOUR_API_KEY"},
  json={"model": "gpt-4", "messages": [{"role": "user", "content": "こんにちは"}]}
)
print(response.json())

いつ使うか：Slackに通知を送る、OpenAIで文章を生成する、freeeに経理データを登録するなど、外部のサービスとデータをやり取りしたいときに使用します。

💡 参考：その他の外部通信系API
REST API以外にも、必要なデータだけを柔軟に取得できるクエリ言語ベースの「GraphQL」や、リアルタイムな双方向通信に特化した「WebSocket」といった技術もあります。これらも「外部と通信して機能を利用する」という点では同じ系統のAPIと言えます。

3. 区分③ Webhook型

向きが逆！相手から通知を受け取る

概念：外部API型とは「通信の向きが逆」になります。自分からリクエストを送るのではなく、こちらが受け皿となるURL（エンドポイント）を公開しておき、外部サービスで何かが起きたときにPush（通知）を受け取る仕組みです。

  • 特徴：イベント駆動型です（例：「メッセージが投稿された時」だけ発火する）。
  • メリット：こちらから定期的に「新しいデータある？」と聞きに行く（ポーリング）必要がありません。
外部サービス ── [イベント発生時にPush送信] ──▶ 自分のURL（受信） ──▶ 処理実行

Slackでのイメージ例：
Slack上でユーザーがメッセージを書き込んだり、リアクションを付けたりする（イベント発生）と、Slack側からあなたの設定したURL宛に「こんな書き込みがありましたよ！」というPOSTリクエストが飛んできます。

GAS の例：Webhookを受信する（doPost関数）
// このスクリプトをWebアプリとして公開するとURLが発行される
function doPost(e) {
  const data = JSON.parse(e.postData.contents);  // 受け取ったデータを解析
  const message = data.text;                      // メッセージを取得
  
  // スプレッドシートに記録する
  SpreadsheetApp.getActiveSheet()
    .appendRow([new Date(), message]);
  
  return ContentService.createTextOutput("OK");
}

いつ使うか：「誰かがフォームを送信した時」「GitHubにコードがPushされた時」「チャットに特定のキーワードが書き込まれた時」など、外部で起きたアクションをトリガー（引き金）にして、自分のプログラムを動かしたいときに使います。

4. 比較まとめ表

ここまでの3つの区分を、それぞれの特徴で比較してみましょう。

区分 HTTP通信 通信の向き 設定難易度 代表例
ライブラリ/SDK型 不要 ―（コード内で完結） ★☆☆ 低 pandas, SpreadsheetApp
外部API型 必要 自分 → 外部サービス ★★☆ 中〜高 Slack API, OpenAI API
Webhook型 必要（受信） 外部サービス → 自分 ★★☆ 中 Slack Webhook, GitHub Webhook

5. 開発をスムーズに進めるためのTIPS

💡 TIP 1: 迷ったらまず「ライブラリ型」で済まないか考えよう プログラムで新しい処理を作りたいとき、まずは「今使っている環境（GASの組み込みサービスや、Pythonの標準ライブラリなど）で解決できないか？」を確認しましょう。通信の設定や認証が不要なため、一番とっつきやすくバグも起きにくい王道のアプローチです。
💡 TIP 2: 外部API型とWebhookは組み合わせて使うことが多い 実際のシステム開発では、この2つをセットで使うことがよくあります。
【例：Slackボットの開発】
① ユーザーからのメンションを受け取る ＝ Webhook型（Slack → 自分）
② 受け取った内容にAIで返信する ＝ 外部API型（自分 → Slack API）
このように両方を組み合わせることで、双方向の豊かなコミュニケーションが実現できます。
