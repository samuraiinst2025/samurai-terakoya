Claude Code Hooks 自動化実践ガイド

イベント駆動で開発ワークフローを完全自動化するための指南書

1. Hooks とは？ ― Webhook と同じ「センサー」機能

Hooks（フック）は、Claude Code 内で特定のイベント（ファイルの編集、タスクの完了など）が発生した際に、任意のシェルコマンドを自動的に実行させる機能です。

🔗 Web 開発者への直感的な例え Hooks は Web 開発における **Webhook** と全く同じ概念です。
「GitHub でプッシュされたらデプロイを走らせる」のと同様に、Claude Code で「ファイルが編集されたら整形を走らせる」という設定が可能です。

手動で行っていた定型作業を Claude に肩代わりさせることで、高品質なアウトプットを維持し続けることができます。

2. 🤔「スキル (Skills)」と何が違うのか？

Claude Code には「スキル」という機能もありますが、最大の違いは **「きっかけ（誰がいつ動かすか）」** です。

比較項目 スキル (Skills) フック (Hooks)
イメージ 「必殺技・マニュアル」 「センサー・罠」
トリガー 能動的（意思がある時）
ユーザーや AI が「やるぞ」と決めた時		 受動的（出来事ベース）
特定のイベントが起きた瞬間に勝手に
典型的活用 複雑なタスクの標準手順化
(例: /deploy コマンド)		 毎回必ず発生する定型処理
(例: 保存時の Linter 実行)

3. シェルの枠を超えた活用：スクリプトへの委譲

Hooks の設定はシェルコマンド形式で記述しますが、これは単なる **「起動スイッチ」** です。Claude にファイル操作ツールがあるならシェルは不要に思えますが、実はシェル経由にすることで大きな自由度が手に入ります。

🐍 シェルを「ランチャー」にする Hooks で複雑なロジックを直接書く必要はありません。
"onTaskCompleted": ["python3 notify.py"]
のように、Python や Node.js、Mac の AppleScript などを呼び出すことで、OS 全体を巻き込んだ高度な自動化が可能です。
  • OS レイヤーの操作: Mac の通知センターへの表示、デスクトップアプリの起動。
  • バイナリ操作: Docker や Git、Terraform といった CLI ツールの確実な実行。
  • 外部サービス連携: curl 一発での Slack/LINE 通知送信。

4. 設定の実践例

設定はプロジェクト内の .claude/hooks.json に記述します。

【例 1】 保存時の自動整形 (Prettier)

{
  "onFileEdited": [
    "npx prettier --write ${changedFile}"
  ]
}

【例 2】 タスク完了時の Slack 通知 (curl 活用)

{
  "onTaskCompleted": [
    "curl -X POST -H 'Content-type: application/json' --data '{\"text\":\"✅ タスク完了\"}' https://hooks.slack.com/services/..."
  ]
}

5. まとめ: 「三種の神器」を使いこなす

「担当者を配する **サブエージェント**」「道具を繋ぐ **MCP**」「自動検知の **Hooks**」こそが、Claude Code の三種の神器です。

これらを組み合わせ、「フックで検知」→「サブエージェントが思考」→「スキルで実行」という黄金のワークフローを構築しましょう。
