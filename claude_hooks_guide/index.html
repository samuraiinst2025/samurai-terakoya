Claude Code Hooks 自動化実践ガイド

イベント駆動で開発ワークフローを完全自動化するための指南書

1. Hooks とは？

Hooks（フック）は、Claude Code 内で特定のイベント（ファイルの編集、タスクの完了など）が発生した際に、任意のシェルコマンドを自動的に実行させる機能です。

手動で行っていた「コードの整形」「テストの実行」「通知」などを Claude に肩代わりさせることで、開発のテンポを一切崩さずに高品質なアウトプットを維持できます。

🤖 Hooks で実現できること
  • ファイルを保存するたびに Prettier で自動整形する
  • タスクが完了したら Slack や LINE に通知を送る
  • 外部のセキュリティスキャンツールを自動起動する
  • 重大なコマンドが提案された時に追加の検証を行う

2. 主要なイベント（トリガー）

現在サポートされている、4つの主要なフックイベントを紹介します。

イベント名 発生タイミング 活用例
onFileEdited Claude がファイルを書き換えた直後 自動フォーマット、シンタックスチェック
onTaskCompleted 会話のタスク（User Request）が完了した時 ビルド、デプロイ準備、完了通知
onInputRequired Claude がユーザーの入力を待機し始めた時 デスクトップ通知、スマホへのプッシュ通知
onCommandProposed Claude がシェルコマンドを実行しようとした時 セキュリティスキャン、コマンドの自動レビュー

3. 設定の実践例

設定は、プロジェクト内の .claude/hooks.json に記述するのが一般的です。また、CLAUDE.md で定義することも可能です。

【実践例 1】 保存時の自動整形 (Prettier)

ファイルが編集されるたびに、プロジェクトのルールに従って綺麗に整頓します。

{
  "onFileEdited": [
    "npx prettier --write ${changedFile}"
  ]
}

【実践例 2】 タスク完了時の Slack 通知

長いタスクが終わった時に、席を外していても通知を受け取れるようにします。

{
  "onTaskCompleted": [
    "curl -X POST -H 'Content-type: application/json' --data '{\"text\":\"✅ タスクが完了しました！\"}' https://hooks.slack.com/services/..."
  ]
}
💡 環境変数の活用 ${changedFile} のような特別な変数をフック内で使用でき、どのファイルが変更されたのかなどをコマンドに渡すことができます。

4. まとめ: 「考えなくていい」開発環境へ

フックを使いこなすことで、開発者は「ツールを動かす」という作業から解放され、「ロジックを考える」ことに集中できます。

「サブエージェント（担当者の配置）」「MCP（外部ツールの接続）」、そしてこの「Hooks（プロセスの自動化）」を組み合わせることが、Claude Code を使いこなす三種の神器となります。
