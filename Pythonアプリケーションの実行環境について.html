レベル 2

小〜中規模バッチ：サーバー/PCで定期実行

既存のサーバーやPCを使って、決まった時間に自動でプログラムを動かす方法です。
ここで最も重要なのが「仮想環境 (venv)」です。

🔰 なぜ「仮想環境」が必要なの？

1つのサーバーで複数のPythonアプリを動かすとき、ライブラリのバージョンが喧嘩（競合）してしまうのを防ぐためです。

❌ 仮想環境なし
既存システムA
pandas 1.5
あなたの新作アプリ
pandas 2.0が必要...入らない！💥

⚠ ライブラリを上書きすると、既存システムが動かなくなる！

✅ 仮想環境あり (venv)
環境 A 既存システムA
pandas 1.5
環境 B あなたの新作アプリ
pandas 2.0

✨ 箱（環境）が分かれているから、お互い干渉しない！

🛠️ 実践！venv の使い方ステップバイステップ

Windows環境を例にした、最も基本的なコマンド手順です。

1仮想環境を作成する

プロジェクトのフォルダに移動して、魔法のコマンドを打ちます。

cd my_project
python -m venv venv
※ 最後の venv は作成するフォルダ名です。

2仮想環境を有効化（Activate）する

これを忘れると意味がありません！

# Windowsの場合
venv\Scripts\activate

# Mac/Linuxの場合
source venv/bin/activate

成功すると、コマンドラインの先頭に (venv) と表示されます。

3ライブラリを入れる＆記録する

# 必要なものをインストール
pip install pandas numpy

# インストールしたリストを保存（納品用）
pip freeze > requirements.txt

4使い終わったら無効化

deactivate

📦 納品時のテンプレート

お客様に渡すフォルダ構成のイメージです。venv フォルダ自体は渡しません！（お客様の環境で作ってもらいます） 

📁 my_project/
├── main.py              # プログラム本体
├── requirements.txt     # 📜 ライブラリの設計図（必須！）
├── run.bat              # 🏃‍♂️ 実行用スクリプト
└── README.md            # 📖 手順書
💡 トラブルシューティング（クリックして開く）
  • Q. pip install でエラーが出る
    A. pip install --upgrade pip でpip自体を更新してみてください。
  • Q. 実行ポリシーエラー（Windows）
    A. PowerShellで実行できない場合、コマンドプロンプトを使うか、ポリシー設定を変更する必要があります。
  • Q. 本番で動かない
    A. requirements.txt を作り忘れていませんか？開発環境で必ず pip freeze しましょう。
レベル 3

ちゃんと運用：ジョブ管理ツール

「Aの処理が終わったらBを実行」「失敗したら通知」など、複雑な管理が必要な場合。

  • 🤖 Airflow / Prefect などのツールを使用。
  • ✅ メリット： 実行履歴が見える、失敗時のリカバリーが楽。
  • ❌ デメリット： ツールの学習コストがかかる。
レベル 4

本格運用：Docker（ドッカー）

「環境ごと箱詰めして配る」技術です。

🗣️ 従来：「Pythonのバージョンが合わなくて動きません…」
🐳 Docker：「このコンテナ（箱）を起動すれば、私のPCと全く同じ環境で動きます！」

  • ✅ 最強の再現性： OSやライブラリの違いによるトラブルが激減。
  • 🛠️ 納品物： Dockerfile + docker-compose.yml
レベル 5

大規模：Kubernetes（クーベネティス）

大規模なWebサービスや、絶対に止めてはいけないシステム向け。
※初学者のうちは「そういうものがある」と知っておくだけでOKです。

📊 規模を決める5つの判断軸

判断軸 小規模（レベル1-2） 大規模（レベル4-5）
実行頻度 手動 / 週1回 常時稼働 / 毎分
利用者数 自分 / チーム数名 全社員 / 外部顧客
失敗時の影響 やり直せばOK 業務停止・損害発生
証跡（ログ） 簡易的でOK 厳密な監査が必要
