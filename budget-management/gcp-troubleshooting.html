🆘 トラブルシューティング

よくあるエラー・問題と解決方法

GCP版ガイドに戻る

⚙️ セットアップ時のエラー

❌ Error: User does not have permission 'iam.serviceAccounts.create'
原因：

GCP プロジェクトの IAM 権限が不足しています。

解決方法：
  1. GCP コンソール → IAM と管理 → IAM を開く
  2. 使用している Google アカウントを検索
  3. 「オーナー」または「エディタ」ロールを付与
  4. terraform apply を再実行
❌ Error: Error creating database: googleapi: Error 400
原因：

同じプロジェクトに既に Firestore データベースが存在しています。

解決方法：
  1. GCP コンソール → Firestore を開く
  2. 既存のデータベースを確認
  3. テスト用なら削除（本番なら以下のステップ）
  4. Terraform コード内のデータベースリソースで depends_on を利用し、作成順序を制御
例
resource "google_firestore_database" "database" {
  ...
  depends_on = [google_firebase_project.default]
}
❌ Error: Terraform plan succeeds but apply fails
原因：

API がまだ有効化されていない、または IAM 権限の反映に時間が必要な場合があります。

解決方法：
1️⃣ API 有効化を確認

GCP コンソール → API とサービス → ライブラリ で以下の API を検索・有効化：

  • Cloud Firestore API
  • Cloud Functions API
  • Cloud Build API
  • Secret Manager API
2️⃣ 数分待機してから再実行

API 有効化と IAM 権限の反映に数分かかることがあります。

sleep 300  # 5分待機
terraform apply
3️⃣ それでもダメなら Cloud Shell で実行

GCP Cloud Shell から実行すれば、認証情報の問題を回避できます。

🚀 実行時のエラー

❌ Firestore セキュリティルール拒否エラー
エラーメッセージ例：
Permission denied. (read_document [root/companies/...])
原因：

Firestore セキュリティルールが正しく設定されていない、またはユーザーが正しい企業に属していない。

解決方法：
  1. Firebase コンソール → Firestore → ルール を確認
  2. Firestore Emulator Suite でルールをテスト
  3. Firestore のテストモードを有効化（本番前のデバッグ用）
Firestore Emulator 実行
firebase emulators:start --only firestore
❌ Cloud Functions でタイムアウト
原因：

Freee API 呼び出しに時間がかかっている、またはデフォルトのタイムアウト（60秒）では不足。

解決方法：
1️⃣ タイムアウト値を増やす
Terraform
resource "google_cloudfunctions_function" "freee_sync" {
  ...
  timeout = 540  # 最大540秒（9分）
}
2️⃣ メモリを増やす

メモリが少ないと CPU が制限され、処理が遅くなります。

Terraform
available_memory_mb = 512  # 256MB から 512MB に増加
3️⃣ Freee API 呼び出しの最適化

不要なデータ取得を避ける、バッチ処理を活用。

❌ Secret Manager へのアクセス失敗
エラーメッセージ例：
Error: Permission denied on resource:
projects/[PROJECT]/secrets/freee-api-token/versions/latest
解決方法：
1️⃣ Cloud Functions の Service Account に権限を付与
Terraform
resource "google_project_iam_member" "functions_secret_accessor" {
  project = var.project_id
  role    = "roles/secretmanager.secretAccessor"
  member  = "serviceAccount:${google_service_account.cloud_functions.email}"
}
2️⃣ 権限の反映を待つ

IAM 権限の反映に数分かかることがあります。

⚡ パフォーマンス最適化

🚀 Cloud Functions を高速化

  • メモリ割り当て： 256MB から 512MB～ に増加（CPU も比例）
  • コールドスタート対策： 定期的に実行してウォーム状態を保つ
  • 並列処理： Freee API 複数呼び出しを Promise.all() で並列化
  • キャッシング： 変更なしのデータは Firestore にキャッシュ

🔍 Firestore クエリ最適化

  • インデックス作成： 頻繁に使用するクエリにはインデックスを作成
  • ページネーション： 大量データは limit() を使ってページ分割
  • 複合クエリ削減： where 条件を少なくする
  • リアルタイムリスニング削減： 必要な時だけ .onSnapshot() を使用

📊 Cloud Logging でパフォーマンス監視

  • Cloud Functions の実行時間を記録
  • Firestore の読み書き数を監視
  • エラーレートをアラートで通知

💰 コスト最適化

💳 GCP コスト削減のポイント

  • Cloud Scheduler 実行回数： 必要最小限の頻度に設定（1日1回など）
  • Cloud Functions メモリ： 必要最小限の割り当て（ただし性能とのバランス）
  • Firestore インデックス： 不要なインデックスを削除
  • ストレージ： 古いログは自動削除ポリシーを設定

月額費用の見積もり例

サービス 設定 月額目安
Cloud Functions 256MB × 1日1回 × 100秒 ¥100～
Cloud Scheduler 1日1回 × 30日 ¥10 × 3 = ¥30
Firestore 1000読み取り / 日 + ストレージ 1GB ¥30～50
合計 ¥150～200/月

❓ よくある質問

Q: Terraform destroy でリソース削除できない
A: Firebase プロジェクトは削除に制限があります。GCP コンソール → Firebase プロジェクト設定 → プロジェクトを削除 から手動で削除してください。
Q: Freee API トークンの有効期限は？
A: Freee API トークンは有効期限が設定されている場合があります。期限切れの場合は Freee 管理画面で新規発行し、Secret Manager を更新してください。
Q: 複数の GCP プロジェクトをセットアップしたい
A: terraform.tfvars で project_id を変更すれば、同じコードで複数プロジェクトをセットアップできます。または環境ごとに別ディレクトリを作成し、バックエンド（terraform state）を分離するのがお勧めです。
Q: Firestore のテストモードと本番モードの違いは？
A: テストモードではセキュリティルール関係なくすべてのアクセスが許可されます（開発用）。本番環境では必ず本番モードに切り替え、厳密なセキュリティルールを設定してください。
Q: ログが見当たらない
A: GCP コンソール → ログエクスプローラ で「リソースタイプ: Cloud Function」や「リソースタイプ: Firestore」でフィルタリングして検索してください。

📞 困った時の対応

  1. ドキュメントを確認
    このガイドやGCP公式ドキュメントを再確認
  2. ログを確認
    Cloud Logging でエラーメッセージの詳細を確認
  3. 単純化してテスト
    複雑な設定を一度削除して、最小限の構成でテスト
  4. 社内チーム・コミュニティに相談
    GCP公式フォーラム、Stack Overflow、Slack コミュニティ等
  5. GCP サポートに問い合わせ
    GCP 有償サポートがあれば、プロフェッショナルサポートに相談可能
