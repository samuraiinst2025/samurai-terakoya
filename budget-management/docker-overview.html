📅 5日間の開発フロー

1

Day 1-2: Docker化（2日）

既存の React コードとバックエンド（Cloud Functions）を Docker コンテナ化します。

  • Dockerfile 作成（フロントエンド用）
  • バックエンド（Node.js/Python）の Docker化
  • ローカルでの動作確認
2

Day 2-3: docker-compose 設定（1.5日）

複数のコンテナを一括管理するための docker-compose.yml を作成。

  • docker-compose.yml の作成
  • .env テンプレート（Freee API トークン等）
  • ネットワーク・ボリューム設定
  • フルテスト（顧客環境を想定）
3

Day 4-5: ドキュメント・検証（1.5日）

顧客向けセットアップガイド作成と実装検証。

  • 「5分で起動するクイックスタート」作成
  • .env の設定方法ガイド
  • よくある質問と回答
  • Docker Hub への push
  • 初回テスト顧客での実施

💡 Docker版採用時の推奨事項

顧客側での準備

  • Docker & Docker Compose インストール
  • サーバー環境の確保 - VPS または オンプレミス
  • Freee API トークン - 事前に取得・用意
  • ネットワーク設定 - ファイアウォール設定確認

自社側での準備

  • Docker スキル - チーム内での習得
  • Docker Hub 管理 - イメージの配布・管理
  • サポート体制 - 初回セットアップ対応者の確保
  • テスト環境 - 自社テスト用の Docker 環境
