🎯 なぜこれが必要か？
Dockerfile と docker-compose を一度作成すると、2社目以降の顧客セットアップが大幅に高速化されます。 毎回ゼロから作成する必要がなく、テンプレートを少し修正するだけで、 異なる顧客環境にも同じコンテナを展開できます。つまり、初回の投資が後々の効率化に繋がります。

💡 Docker 運用のベストプラクティス

イメージのバージョン管理

Docker イメージにはバージョンタグを付け（v1.0.0 など）、更新履歴を明確にします。

機密情報の安全管理

.env ファイルは Git に含めず、.gitignore に追加します。 本番環境ではファイルパーミッション（chmod 600）で保護します。

ログの一元管理

Docker は stdout/stderr へのログ出力を基本とし、json-file ドライバーでローテーション設定を行います。

定期的なイメージ更新

Node.js やライブラリのセキュリティパッチが公開されたら、イメージを再ビルドして新しいバージョンを配布します。
