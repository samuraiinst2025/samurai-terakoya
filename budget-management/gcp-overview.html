📋 GCP版の全体像

📅 7日間の開発フロー

1

Day 1-2: Terraform スクリプト作成（2日）

GCP 自動セットアップスクリプトを作成します。

  • Firebase プロジェクト構築の Terraform コード作成
  • Cloud Functions、Firestore の自動初期化
  • Secret Manager、IAM ルール設定
  • 変数テンプレート作成と ドライラン・テスト
2

Day 3-4: Firebase 設定・デプロイ（2日）

Firebase サービスの詳細設定とデプロイパイプラインを構築します。

  • Firebase Hosting の設定と React ビルド環境
  • Cloud Functions の最適化・ロギング強化
  • Firestore セキュリティルール作成
  • 本番環境シミュレーションテスト
3

Day 5-7: ドキュメント・検証（2-3日）

顧客向けドキュメント作成と実装検証を行います。

  • 顧客向けセットアップガイド作成
  • GCP IAM ロール設定テンプレート作成
  • サンドボックス環境での実装テスト
  • CI/CD パイプライン整備

💡 GCP版採用時の推奨事項

顧客側での準備

  • GCP アカウント - 新規または既存の GCP プロジェクト
  • IAM 権限 - プロジェクトオーナー権限が必要
  • Freee API トークン - 事前に取得・用意
  • 技術サポート体制 - GCP 関連のサポート窓口の確認

自社側での準備

  • Terraform スキル - チーム内での習得
  • GCP 運用知識 - 監視・ロギング・コスト管理
  • サポート体制 - 初回セットアップ対応者の確保
  • テスト環境 - 顧客テスト用の GCP サンドボックス

❓ よくある質問

Q: Terraform を学ぶのにどのくらい時間がかかりますか？

A: 基本的な使い方であれば 2-3日で習得可能です。このガイドの「Terraform ガイド」で実装例を示しているので、そこから開始できます。

Q: 顧客が GCP を持っていない場合はどうすればいい？

A: セットアップガイドで「GCP アカウント作成手順」を提供します。初回セットアップ時に一緒に作成をサポートするのが一般的です。

Q: GCP の課金が想定より高くなったら？

A: Cloud Functions のメモリ設定やスケジューリング最適化で削減できます。詳しくは「トラブルシューティング」ページのコスト最適化セクションをご覧ください。

Q: Docker版から GCP版に移行できますか？

A: 可能です。Firestore のデータはエクスポート・インポート機能で移行できます。詳しくは自社に お問い合わせください。
