Firebase で作ったアプリをオンプレミスで動かすには

📋 ガイドトップ > Firebase → Docker 変換

🎯 何を変換するのか

Firebase で作られたアプリケーションは、クラウド上の複数のサービスで構成されています。 これをオンプレミス環境（Docker）で動かすには、各コンポーネントの役割を理解して、 適切に「置き換える」必要があります。

🔄 変換のイメージ

Firebase版（クラウド）

  • Firebase Hosting
    (React UI ホスト)
  • Cloud Functions
    (API サーバー)
  • Firestore
    (データベース)
  • Firebase Auth
    (認証)

Docker版（オンプレ）

  • Express.js
    (フロント + API)
  • Express.js
    (既に含まれる)
  • Firestore SDK 継続
    (クラウド接続）
  • Firebase SDK 継続
    (クラウド接続)
💡 重要なポイント
**すべてを置き換える必要はありません。** Firebase が提供していた「ホスティング」と「サーバー機能」を Docker で自分たちでやるだけです。 データベースと認証は引き続き Firebase を使うのが一般的です。

Cloud Functions → Express.js への変換

Firebase の Cloud Functions（サーバーレス関数）は、 Express.js（通常の Node.js サーバー）で置き換えます。

Firebase Cloud Functions の例 // 元の Cloud Functions exports.syncFreeeData = functions.https.onRequest(async (req, res) => { const data = await getFreeeData(); await db.collection('budgets').doc(req.body.id).set(data); res.json({success: true}); });
↓ Docker で実行する場合
Express.js で書き換えた例 // Express.js サーバー内 app.post('/api/sync-freee', async (req, res) => { const data = await getFreeeData(); await db.collection('budgets').doc(req.body.id).set(data); res.json({success: true}); }); app.listen(5000, () => { console.log('Server running on port 5000'); });
📝 変換ポイント
• `functions.https.onRequest` → `app.post` に変更
• ロジック自体（getFreeeData など）は同じ
• Firebase SDK の使い方も変わらない
• Docker 起動時に自動的にサーバーが立ち上がる

📱 React UI（画面）はどうするか

Firebase Hosting から Docker へ

項目 Firebase版 Docker版
ビルド npm run build で生成 npm run build で生成（同じ）
デプロイ firebase deploy Docker イメージに含める
ホスト Firebase Hosting Express.js が静的ファイル提供
開発中 npm start で localhost:3000 npm start で localhost:3000（同じ）
✅ 朗報
React の UI コード自体は **一切変わりません。**
ビルド後の静的ファイル（HTML, CSS, JS）を Express.js で配信するだけです。

🚀 全体フロー（シンプル版）

STEP 1: React UI をビルド
npm run build → dist/ フォルダ生成
STEP 2: Express.js サーバー作成
ビルド済み UI + API エンドポイント
STEP 3: Dockerfile で Docker 化
Node.js + Express + React UI をコンテナに詰める
STEP 4: docker-compose で運用
顧客環境で起動 → ブラウザで http://localhost:5000 にアクセス

📌 まとめ

  • ✅ React UI コード：変わらない
  • ✅ ビジネスロジック：ほぼ変わらない（関数型 → Express ルート）
  • ✅ Firebase SDK：変わらない（引き続き使用）
  • 🔄 デプロイ方法：Firebase デプロイ → Docker コンテナ起動

つまり、Firebase で動くアプリケーションは、
ロジックを少し整理して Express.js + Docker でパッケージすれば、 簡単にオンプレミス化できるということです。
