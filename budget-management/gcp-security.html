🎯 セキュリティの3層

GCP での予算管理システムのセキュリティは、以下の3層で構成されます：

1️⃣ 認証層（Authentication）

「ユーザーは本当に本人か？」を確認

  • Firebase Authentication でユーザー確認
  • JWT トークンによる API 認証

2️⃣ 認可層（Authorization）

「このユーザーは何ができるか？」を制御

  • Firestore セキュリティルール
  • GCP IAM ロール

3️⃣ データ暗号化層

「データを安全に保つ」

  • Secret Manager で秘密情報管理
  • 通信の HTTPS 暗号化
  • 保存時の自動暗号化

📚 Firestore セキュリティルール

Firestore データへのアクセスを制御する最も重要な設定です。

基本的なルール構造

firestore.rules
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // ここにルールを記述
  }
}

企業ごとのデータ分離（推奨パターン）

firestore.rules - テナント分離
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 企業ドキュメント
    match /companies/{companyId} {
      // 自分の企業データのみアクセス可能
      allow read, write: if request.auth.uid != null
        && getUserCompany(request.auth.uid) == companyId;

      // 予算データ
      match /budgets/{budgetId} {
        allow read: if request.auth.uid != null;
        allow write: if request.auth.uid != null
          && isAdmin(request.auth.uid);
      }

      // 実績データ
      match /actuals/{actualId} {
        allow read: if request.auth.uid != null;
        allow write: if false; // 読み取り専用
      }
    }
  }

  // ヘルパー関数
  function getUserCompany(uid) {
    return get(/databases/$(database)/documents/users/$(uid)).data.company_id;
  }

  function isAdmin(uid) {
    return get(/databases/$(database)/documents/users/$(uid)).data.role == 'admin';
  }
}

🔒 セキュリティのポイント

  • ✓ 企業 ID で必ずデータ分離
  • ✓ 読み取り・書き込みを明確に分ける
  • ✓ 管理者のみが書き込める操作を制限
  • ✓ Firestore Simulator で事前テスト

🔑 Secret Manager での秘密情報管理

Freee API トークン、Google API キーなどの秘密情報は Secret Manager で管理します。

❌ やってはいけないこと

❌ NG: terraform.tfvars に記載

freee_api_token = "xxxxx"
# ↑ Git にコミットされる
#   リポジトリ履歴から削除不可

❌ NG: ソースコードに埋め込み

const FREEE_TOKEN = "xxxxx";
// ↑ コード履歴に永続化

✅ やるべきこと

✅ OK: Secret Manager

// Terraform で管理
resource "google_secret_manager_secret" "freee_token" {
  secret_id = "freee-api-token"
}

resource "google_secret_manager_secret_version" "freee_token" {
  secret      = google_secret_manager_secret.freee_token.id
  secret_data = var.freee_api_token
}

✅ OK: 環境変数

export TF_VAR_freee_api_token="xxxxx"
# ↑ ターミナルセッション内でのみ有効
#   .bash_history にも記録されない設定可

Cloud Functions での Secret Manager 利用

Cloud Functions - Secret Manager 参照
const secretmanager = require('@google-cloud/secret-manager');

async function getFreeeToken() {
  const client = new secretmanager.SecretManagerServiceClient();
  const name = client.secretVersionPath(
    process.env.GCP_PROJECT_ID,
    'freee-api-token',
    'latest'
  );

  const [version] = await client.accessSecretVersion({name});
  return version.payload.data.toString();
}

👥 GCP IAM ロール設定

Cloud Functions やその他のサービスに最小限の権限を付与します。

推奨される IAM ロール割り当て

サービス ロール 説明
Cloud Functions Cloud Datastore User Firestore 読み書き権限
Cloud Functions Secret Accessor Secret Manager の秘密にアクセス
Cloud Functions Logs Writer ログ出力権限
Firebase プロジェクト Firebase Viewer Firebase リソースの参照権限

Terraform での IAM ロール設定

Terraform - IAM
resource "google_service_account" "cloud_functions" {
  account_id   = "budget-app-functions"
  display_name = "Budget App Cloud Functions"
}

# Firestore 読み書き権限
resource "google_project_iam_member" "cloud_functions_datastore" {
  project = var.project_id
  role    = "roles/datastore.user"
  member  = "serviceAccount:${google_service_account.cloud_functions.email}"
}

# Secret Manager アクセス権限
resource "google_project_iam_member" "cloud_functions_secret" {
  project = var.project_id
  role    = "roles/secretmanager.secretAccessor"
  member  = "serviceAccount:${google_service_account.cloud_functions.email}"
}
⚠️ 重要：最小権限の原則
必要な権限のみを付与してください。Editor や Owner 権限は避けましょう。

📋 監査・ロギング設定

Cloud Logging の有効化

Cloud Functions と Firestore のアクティビティをログに記録し、問題発生時の追跡を可能に。

Cloud Functions - ログ出力例
const functions = require('@google-cloud/functions-framework');
const logger = require('@google-cloud/logging').v2.LoggingServiceV2Client;

functions.http('syncFreeeData', (req, res) => {
  console.log('Freee sync started at ' + new Date());

  try {
    // 処理
    console.log('Successfully synced data');
    res.status(200).send('Sync completed');
  } catch (error) {
    console.error('Error during sync:', error);
    res.status(500).send('Sync failed');
  }
});
