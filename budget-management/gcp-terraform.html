📖 Terraform とは？

Terraform は HashiCorp が開発した Infrastructure as Code（IaC） ツールです。クラウドリソース（GCP、AWS、Azure など）の構成をコードで管理できます。

Terraform を使うメリット

  • 再現性： 同じコードを実行すれば、同じ構成が自動に作成される
  • 管理性： すべてのリソースをテキストファイルで管理・バージョン管理可能
  • 効率性： 手作業を削減し、数分で複雑なセットアップが完了
  • チーム開発： コードレビュー → 本番反映という流れが可能

Terraform の基本概念

用語 説明
Provider クラウドプロバイダー（GCP、AWS等）との連携を定義
Resource 作成・管理するリソース（Firestore、Cloud Functions等）
Variable 設定値を変数化（プロジェクト ID、リージョン等）
Output 実行後の出力値（作成されたリソースの ID 等）
State 現在のリソース状態を記録するファイル

📁 プロジェクトの構成

Terraform プロジェクトの推奨ファイル構成：

budget-management-gcp/
├── main.tf
├── variables.tf
├── outputs.tf
├── terraform.tfvars
└── .gitignore

各ファイルの役割

📄 main.tf

メインのリソース定義ファイル。Firebase プロジェクト、Firestore、Cloud Functions などを記述します。

📄 variables.tf

変数定義ファイル。プロジェクト ID やリージョンなど、環境によって変わる値を変数として定義します。

📄 outputs.tf

出力値定義ファイル。実行後に表示する情報（作成されたリソースのエンドポイント等）を定義します。

📄 terraform.tfvars

変数の具体値を設定するファイル。環境ごとに異なる値（本番環境、テスト環境等）を記載します。

📄 .gitignore

Git にコミットしてはいけないファイルを指定（terraform.tfstate、.tfvars など）

💻 実装例: main.tf

Terraform & Provider 設定

main.tf - Terraform設定
terraform {
  required_version = ">= 1.0"
  required_providers {
    google = {
      source  = "hashicorp/google"
      version = "~> 5.0"
    }
  }
}

provider "google" {
  project = var.project_id
  region  = var.region
}

Firebase プロジェクト作成

main.tf - Firebase
resource "google_firebase_project" "default" {
  project = var.project_id
  provider = google
}

Firestore データベース作成

main.tf - Firestore
resource "google_firestore_database" "database" {
  project     = var.project_id
  name        = "budget-app"
  location_id = var.region
  type        = "FIRESTORE_NATIVE"

  depends_on = [google_firebase_project.default]
}

Cloud Functions 作成

main.tf - Cloud Functions
resource "google_cloudfunctions_function" "freee_sync" {
  name        = "freee-sync-function"
  runtime     = "nodejs20"
  region      = var.region
  description = "Freee data synchronization function"

  available_memory_mb = 256
  source_archive_bucket = google_storage_bucket.function_source.name
  source_archive_object = google_storage_bucket_object.function_zip.name

  entry_point = "syncFreeeData"

  environment_variables = {
    FIRESTORE_PROJECT_ID = var.project_id
    FREEE_API_TOKEN      = var.freee_api_token
  }

  service_account_email = google_service_account.cloud_functions.email
}
💡 ポイント： Freee API トークンなどの秘密情報は、環境変数で参照するのではなく Secret Manager を使う方がセキュアです。

🔧 実装例: variables.tf

variables.tf
variable "project_id" {
  description = "GCP Project ID"
  type        = string
}

variable "region" {
  description = "GCP Region"
  type        = string
  default     = "asia-northeast1"
}

variable "environment" {
  description = "Environment name"
  type        = string
  validation {
    condition     = contains(["dev", "staging", "production"], var.environment)
    error_message = "Environment must be dev, staging, or production."
  }
}

variable "freee_api_token" {
  description = "Freee API Token"
  type        = string
  sensitive   = true
}
⚠️ 重要： sensitive = true を指定することで、コンソール出力時にトークンが隠されます。

⚙️ 実装例: terraform.tfvars

環境ごとに異なる値を記載します。このファイルは .gitignore に追加し、Git にコミットしないようにしてください。

terraform.tfvars - テスト環境
project_id      = "budget-app-test"
region          = "asia-northeast1"
environment     = "dev"
freee_api_token = "xxxxxxxxxxxxx"
terraform.tfvars - 本番環境
project_id      = "budget-app-production"
region          = "asia-northeast1"
environment     = "production"
freee_api_token = "xxxxxxxxxxxxx"
💡 ポイント： 秘密情報（API トークン）は terraform.tfvars に記載する代わりに、環境変数から読み込む方法もあります：
export TF_VAR_freee_api_token="xxxxx"

📤 実装例: outputs.tf

実行後に表示される出力値を定義します。

outputs.tf
output "firebase_project_id" {
  description = "Firebase Project ID"
  value       = google_firebase_project.default.project_id
}

output "firestore_database_name" {
  description = "Firestore Database Name"
  value       = google_firestore_database.database.name
}

output "cloud_function_name" {
  description = "Cloud Function Name"
  value       = google_cloudfunctions_function.freee_sync.name
}

output "firebase_hosting_url" {
  description = "Firebase Hosting URL"
  value       = google_firebase_web_app.default.default_url
}

🚀 実行コマンド

1. 初期化

terraform init

Terraform の初期化。プロバイダーをダウンロード・セットアップします。初回実行時に1回だけ実行。

2. フォーマットと検証

terraform fmt          # コードをフォーマット
terraform validate    # 構文エラーをチェック

3. プラン確認

terraform plan

実際に実行する前に、どんなリソースが作成・変更されるかを確認します。

4. 本実行

terraform apply

実際にリソースを作成。確認メッセージが出るので「yes」を入力。

5. 削除

terraform destroy

テスト後に全リソースを削除したい場合。

⚠️ 注意： destroy は危険。本番環境では実行しないよう注意。

✨ Terraform ベストプラクティス

1. State ファイル管理

terraform.tfstate は Git にコミットしてはいけません。チーム開発の場合は GCS（Google Cloud Storage）にリモート保存するのがお勧め。

backend.tf
terraform {
  backend "gcs" {
    bucket = "budget-app-terraform-state"
    prefix = "prod"
  }
}

2. 環境の分離

dev、staging、production を separate な directories で管理し、異なる tfvars ファイルを使用。

3. セキュリティ

  • secret 情報は Secret Manager で管理
  • sensitive = true で出力から隠す
  • .gitignore で tfvars ファイルを除外

4. テスト

本番環境に適用する前に、必ず terraform plan で確認・テスト環境で検証。

5. ドキュメント

各変数と出力値に description をつけ、保守性を高める。

🆘 よくあるエラー

❌ Error: User does not have permission

原因： 認証情報の IAM 権限不足

対策： GCP コンソール → IAM & 管理 → ロール編集で必要な権限を追加

❌ Error: Error creating database: googleapi: Error 400

原因： Firestore database が既に存在

対策： GCP コンソール → Firestore で既存 DB を確認・削除

❌ Terraform Plan が成功するが Apply で失敗

原因： API 有効化忘れ、IAM 権限のタイミング問題

対策： terraform apply の数分後に再実行（API 反映待ち）
