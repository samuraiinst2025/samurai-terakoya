Claude Code MCP 実践活用ガイド

外部ツールとデータを Claude Code に統合するための究極のバイブル

1. MCP (Model Context Protocol) とは？

MCP は、Claude Code が外部のデータベース、API、ツールとシームレスに連携するための標準プロトコルです。これにより、Claude は単なる「コード書き」を超えて、実社会のデータやワークフローを操作できる「万能エージェント」へと進化します。

🌟 MCP で広がる可能性
  • JIRA の課題を確認し、実装して PR を作成する
  • Sentry のエラーログを解析し、修正案を提示する
  • Notion や Slack の情報に基づいてドキュメントを更新する
  • データベース (PostgreSQL 等) から直接データをクエリする

2. MCP サーバーの追加方法

Claude Code に MCP サーバーを追加するには、claude mcp add コマンドを使用します。通信方式（トランスポート）に合わせて3つのオプションがあります。

オプション 1: リモート HTTP サーバー

Notion 等のリモートで稼働しているサーバーに接続します。

claude mcp add --transport http notion https://mcp.notion.com/mcp

オプション 2: リモート SSE サーバー

Server-Sent Events を使用したストリーミング接続に対応します。

claude mcp add --transport sse asana https://mcp.asana.com/sse

オプション 3: ローカル stdio サーバー (最も一般的)

ローカルでプログラムを起動して通信します。npm や python を直接叩く場合に使います。

claude mcp add --transport stdio --env API_KEY=xxx my-tool -- npx -y @some/mcp-server
🔧 サーバーの管理コマンド
  • claude mcp list : 設定済みのサーバー一覧を表示
  • claude mcp get <name> : 特定のサーバーの詳細を確認
  • claude mcp remove <name> : サーバーを削除
  • /mcp : (チャット内) ステータス確認と認証の実行

3. MCP 設定のスコープ

設定を保存する場所を --scope フラグで制御できます。誰と、どこでツールを共有したいかによって使い分けます。

スコープ 保存場所 主なユースケース
local (デフォルト) .claude/settings.local.json 個人用、実験的、秘密鍵を含む設定
project .mcp.json チーム共有、プロジェクト固有の共有ツール
user ~/.claude.json 全プロジェクトで共通して使う自分の道具箱

4. 実践的な活用シナリオ

GitHub 連携でのコードレビュー

Copilot 経由の MCP サーバーを接続すると、PR の高度なレビューが可能になります。

claude mcp add --transport http github https://api.githubcopilot.com/mcp/
Prompt: 「PR #456 をレビューして改善を提案してください」

Sentry 連携でのエラー修正

claude mcp add --transport http sentry https://mcp.sentry.dev/mcp
Prompt: 「過去24時間で発生した最も致命的なエラーのスタックトレースを解析して修正案を作って」

データベースの直接操作

PostgreSQL サーバーを接続すれば、スキーマの確認からクエリ実行まで Claude が代行します。

Prompt: 「ENG-4521 機能を試したユーザーのリストを DB から抽出して」

まとめ: MCP で Claude Code を自分色に染める

MCP は Claude Code の限界を突破する魔法のプロトコルです。適切なサーバーを追加し、さらに CLAUDE.md に「必要に応じてこれらのツールを使いなさい」と指示を書いておくことで、真の自律型開発環境が手に入ります。
