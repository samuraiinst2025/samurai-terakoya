📚 Claude Agent SDK の基本概念

Claude Agent SDK とは？

Claude Agent SDK は、Anthropic が提供するツールで、複数の AI エージェント（Claude）を協調させて、複雑なタスクを解決するためのフレームワークです。

簡単に言うと: 「複数の AI が協力して仕事をする」仕組みを簡単に作れるツール

単一エージェント vs マルチエージェント

特性 単一エージェント マルチエージェント
構成 Claude 1個 Claude 複数個（親 + 子エージェント）
適している用途 単純なタスク（翻訳、質問回答） 複雑なタスク（データ入力 + 分析）
実装の複雑さ シンプル 複雑だが、SDK が管理を簡潔に
コスト 低い 高い（複数エージェント × API呼び出し）

📊 予算管理システムでの例

このガイドで構築する「予算管理システム」では、マルチエージェント構成が必要になります：

  • 親エージェント（Budget Orchestrator）: ユーザーのリクエストを受け取り、適切なエージェントに振り分ける
  • 子エージェント 1（InputAssistant）: 取引データの入力サポート、バリデーション、重複検知
  • 子エージェント 2（AnalysisAgent）: 支出パターン分析、予算予測、異常検知

✅ マルチエージェント構造がもたらすメリット:

  • 各エージェントが専門分野に集中 → 精度向上
  • 複雑なタスクが複数の小さなタスクに分割 → 実装が簡潔
  • SDK が通信・エラーハンドリングを自動管理 → 開発が楽

🔑 重要なコンセプト

1. 親エージェント（Orchestrator）

ユーザーのリクエストを分析して、どのエージェントに仕事を任せるか判断する司令官的な存在です。

// 親エージェントのイメージ 「ユーザーが 『新しい取引を追加して』 と言った」 ↓ 分析 「これは入力系のリクエストだ」 ↓ 判定 「InputAssistant エージェントに任せよう」 ↓ 委譲 InputAssistant が動作開始

2. 子エージェント（Sub-agents）

親エージェントから委譲されたタスクに特化したエージェント。各エージェントは独自のツールと指示を持ちます。

3. SDK の自動管理

親と子の間の通信、エラーハンドリング、タイムアウト管理などを SDK が自動で行います。開発者は実装に専念できます。

⚠️ 注意: SDK は子エージェントの子エージェント（3層以上）を許可していません。親 → 子、この 2 層構成で設計してください。

Create actionable recommendations Provide data-driven insights with clear reasoning.

🔐 Step 4: Firestore 連携

ここでは、Firestore データベースの設定とセキュリティルールを実装します。

Firestore コレクション設計

以下の構造で Firestore を設計：

firebase-project/ ├── users/{userId} │ ├── email: string │ ├── name: string │ ├── createdAt: timestamp │ └── preferences: object │ ├── transactions/{transactionId} │ ├── userId: string │ ├── amount: number │ ├── category: string │ ├── date: timestamp │ ├── description: string │ ├── createdAt: timestamp │ └── processingStatus: string │ └── analysis_results/{resultId} ├── userId: string ├── period: string (YYYY-MM) ├── trends: object ├── recommendations: array └── generatedAt: timestamp

セキュリティルール設定

ファイル: firestore.rules

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // ユーザー: 自分のデータのみアクセス可能 match /users/{userId} { allow read, write: if request.auth.uid == userId; } // 取引: 自分のデータのみアクセス可能 match /transactions/{transactionId} { allow create: if request.auth.uid == request.resource.data.userId && request.resource.data.keys().hasAll(['amount', 'date', 'category']); allow read: if request.auth.uid == resource.data.userId; allow update: if request.auth.uid == resource.data.userId && !('userId' in request.resource.data.diff().changedKeys()); allow delete: if request.auth.uid == resource.data.userId; } // 分析結果: 自分のデータのみアクセス可能 match /analysis_results/{resultId} { allow read, write: if request.auth.uid == resource.data.userId; } } }

Firestore ツール実装

ファイル: functions/tools/firestore-tools.js

const admin = require("firebase-admin"); const db = admin.firestore(); // トランザクション保存 async function saveTransaction(userId, transactionData) { const docRef = db.collection("transactions").doc(); await docRef.set({ userId, ...transactionData, createdAt: admin.firestore.FieldValue.serverTimestamp(), processingStatus: "validated" }); return docRef.id; } // 取引をフェッチ（期間指定） async function fetchTransactions(userId, startDate, endDate) { const snapshot = await db .collection("transactions") .where("userId", "==", userId) .where("date", ">=", startDate) .where("date", "<=", endDate) .get(); return snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() })); } // 重複チェック async function checkDuplicates(userId, amount, dateRange = 7) { const now = new Date(); const pastDate = new Date(now.getTime() - dateRange * 24 * 60 * 60 * 1000); const snapshot = await db .collection("transactions") .where("userId", "==", userId) .where("amount", "==", amount) .where("date", ">=", pastDate) .get(); return snapshot.docs.length > 0; } module.exports = { saveTransaction, fetchTransactions, checkDuplicates };

🔧 トラブルシューティング

よくあるエラーと その解決方法を紹介します。

API キーが見つからない

エラー: Error: API key not found. Set CLAUDE_API_KEY environment variable.

原因: 環境変数が設定されていない

解決策:

  1. .env ファイルを確認: CLAUDE_API_KEY=sk-...
  2. ターミナルでリロード: source .env
  3. 設定確認: echo $CLAUDE_API_KEY

Firebase 認証失敗

エラー: Error: User must be authenticated to call this Cloud Function

原因: リクエストに ID Token が含まれていない

解決策:

  1. Firebase Authentication を有効化
  2. クライアント側で ID Token を取得して送信
  3. Cloud Functions でトークンを検証

Firestore セキュリティルール エラー

エラー: Error: Missing or insufficient permissions.

原因: セキュリティルール が存在しないか、権限がない

解決策:

  1. Firestore コンソールでテストモードか本番モードか確認
  2. セキュリティルールが正しく展開されたか確認: firebase deploy --only firestore:rules
  3. userId がリクエストとドキュメントで一致しているか確認

Cloud Functions タイムアウト

エラー: Error: Function execution took 540 seconds and timed out.

原因: エージェント処理が遅い、またはネットワーク待機

解決策:

  1. Cloud Functions のタイムアウトを増やす: timeoutSeconds: 600
  2. エージェント処理を最適化（不要な API 呼び出しを削減）
  3. 非同期処理を導入（Pub/Sub）

エージェント間通信エラー

エラー: Error: Sub-agent invocation failed.

原因: 親エージェントから子エージェントへの通信失敗

解決策:

  1. 親エージェントの agent_id が正しいか確認
  2. 子エージェント が正しく作成されたか確認
  3. SDK のバージョンが最新か確認: npm list @anthropic-ai/sdk
  4. CloudLogging でエラーログを確認

❓ よくある質問

Q1: マルチエージェント構造の利点は？

A: 以下のメリットがあります：

  • 各エージェントが専門分野に集中 → 精度向上
  • 複雑なタスクが小さなタスクに分割 → 実装が簡潔
  • スケーラビリティ向上 → 新しい機能の追加が容易

Q2: SDK が自動管理する「タイムアウト」って何？

A: 子エージェントの処理が 30 秒を超えた場合、SDK が自動的に中断してリトライ（最大 3 回）します。

Q3: エージェントは何層まで作れる？

A: 親 → 子、この 2 層まです。子エージェント が子エージェント（孫エージェント）を作ることはできません。

Q4: 複数ユーザーが同時リクエストしたら？

A: Cloud Functions が自動的にスケーリングして並列処理します。ただし、Claude API のレート制限（例：1分当たりのトークン数）に注意してください。

Q5: コストはどのくらい？

A: Claude API は入出力トークンで課金されます。

  • 月 1,000 リクエスト（平均 2,500 tokens/request）: 約 $80
  • Firebase Firestore: 読み取り/書き込み回数で課金

Q6: ローカルで完全にテストできる？

A: はい。Firebase Emulator を使用すればローカル Firestore でテストできます。ただし、Claude API はクラウドで実行されるため、実際の API 呼び出し環境でのテストが必要です。

Q7: Python で実装する場合、何が変わる？

A: Claude Agent SDK の API は同じです。言語の違いは以下のみ：

  • import 文が異なる
  • 変数の型宣言
  • async/await の構文

アーキテクチャ・ロジックは完全に同じです。

Q8: セキュリティで注意することは？

A: 以下が重要です：

  • API キーを .env で管理（GitHub にコミットしない）
  • Firestore Security Rules で userId チェック
  • ログから個人情報（金額など）を削除またはハッシュ化
  • 本番環境では HTTPS 必須

Q9: エラーが出た時、どこを見る？

A: 以下の順番でログを確認：

  1. ローカル: ターミナル出力
  2. Cloud Functions: firebase functions:log
  3. Google Cloud: Cloud Logging コンソール
  4. Firestore: Firestore コンソール（データ確認）

Q10: 本番環境で何を監視すべき？

A: 以下を監視してください：

  • Cloud Functions の実行時間（> 5秒でアラート）
  • エラーレート（> 5%でアラート）
  • Claude API のトークン使用量（日額上限の 80%でアラート）
  • Firestore の読み取り/書き込み数（クォータの 80%でアラート）
