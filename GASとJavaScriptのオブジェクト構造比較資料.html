GASのオブジェクトとクラス

🚀 通常のJavaScriptとの比較で理解する、GASでのオブジェクト操作の基本

1. 💻 通常のJavaScript：クラスのインスタンス化

JavaScriptで新しいモノ（オブジェクト）を作る時は、設計図（クラス）から new を使って実体（インスタンス）を作ります。

① クラス（設計図）の定義 
class Car {
  constructor(name) {
    this.name = name;
  }
  drive() {
    console.log(this.name + " が走る");
  }
}
② インスタンス化 & 実行 
// new で生成
const car = new Car("プリウス");

// メソッド実行
car.drive();
クラス（Car）
設計図
↓ new
インスタンス（car）
実体化された車
↓
メソッド実行（drive）
車が走る

2. 🍀 GASの場合：new を使いません

GASでは new を使って自分でオブジェクトを作ることはほとんどありません。
代わりに「サービス」と呼ばれる入口から、メソッドをつなげて目的のモノ（シートなど）を取得します。 

const sheet = SpreadsheetApp
    .getActiveSpreadsheet()  // スプレッドシートを取得
    .getActiveSheet();      // その中のアクティブなシートを取得
サービス（SpreadsheetApp）
Google機能への入口
↓ メソッド呼び出し
オブジェクト取得（Spreadsheet）
ファイルそのもの
↓ メソッド呼び出し
オブジェクト取得（Sheet）
特定のシート

3. 🔍 実は裏ではクラスがある

GASにも Spreadsheet や Sheet、Range といったクラスは存在します。
しかし、new Spreadsheet() のように自分で作ることはできません。

なぜ？ 🤔

GASは、Googleがサーバー上に既に持っているデータ（既存のファイルやシート）を操作するための言語だからです。
「新しく作る」のではなく、「そこにあるものを取ってくる（取得する）」という考え方が中心になります。

比較項目 通常 JavaScript GAS (Google Apps Script)
インスタンスの生成方法 new Car() app.getSheet() ※メソッドで取得
クラスの用意 自分でコードを書く Googleが用意してくれている
考え方 自分でゼロから生み出す 既存のデータにアクセスする

5. 📝 実際のコード比較

JavaScript (自分で作る) 
// 1. 生成
const car = new Car("プリウス");

// 2. 操作
car.drive();
GAS (既存データを取得) 
// 1. 取得 (生成ではない)
const sheet = SpreadsheetApp
  .getActiveSpreadsheet()
  .getActiveSheet();

// 2. 操作
sheet.getRange("A1").setValue("Hello");
